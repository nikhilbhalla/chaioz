## Step 1 — One-time setup (~15 min)

```bash
# 1. Install eas-cli globally
yarn global add eas-cli            # or: npm i -g eas-cli

# 2. Sign in with the Chaioz Expo account
eas login                          # opens browser → expo.dev/login

# 3. Initialise the project (only the first time)
cd /app/mobile
eas init --id auto                 # creates a project, fills app.json → extra.eas.projectId

# 4. Configure native credentials (auto-generated by EAS)
eas credentials                    # follow prompts — let EAS manage everything
```

**Key thing to remember:** when `eas init` runs, it automatically writes `extra.eas.projectId` into `app.json`. Commit this change.

---

## Step 2 — First iOS build → TestFlight (~30 min wait)

```bash
cd /app/mobile

# Builds in EAS cloud (no Xcode needed). Picks up your Apple ID via web prompt.
eas build --profile preview --platform ios

# When it finishes, EAS gives you a build URL. To send to TestFlight:
eas submit --profile production --platform ios --latest
```

After `eas submit`, Apple takes ~5–15 min to process. Then:

1. Open **App Store Connect → My Apps → Chaioz → TestFlight**
2. Click the new build → fill in the "Test Information" section (privacy URL, beta description)
3. **Add yourself as an internal tester** (uses your Apple ID — no review needed, instant)
4. Open **TestFlight on your iPhone** → install Chaioz → log in → grant push permission

📝 EAS will print the **Apple Team ID** (looks like `XYZ123ABCD`) — save that, you'll need it in step 4.

## Step 3 — First Android build → Play Console internal track (~20 min wait)

```bash
cd /app/mobile

# Builds an .aab (Android App Bundle) ready for Play Store
eas build --profile preview --platform android

# Submits to Play Console → Internal Testing track
eas submit --profile production --platform android --latest
```

EAS will print the **SHA-256 cert fingerprint** for the Android build (looks like `AB:CD:EF:...`). Save that too.

## Step 4 — Wire universal links (the "https://chaioz.com.au/order/X opens the app" magic)

After step 2 + 3 you have **Apple Team ID** + **Android SHA-256**. Drop them in:

### a) Backend env

```bash
# /app/backend/.env
APPLE_TEAM_ID=XYZ123ABCD
ANDROID_SHA256_FINGERPRINTS=AB:CD:EF:01:23:45:67:89:...
```

Then `sudo supervisorctl restart backend`.

### b) Public well-known files (must be served from chaioz.com.au)

Edit these two files:

```bash
/app/frontend/public/.well-known/apple-app-site-association
/app/frontend/public/.well-known/assetlinks.json
```

Replace `TEAMIDXXXX` and `FILL_AFTER_FIRST_EAS_BUILD` with the real values.

**Then deploy them at chaioz.com.au.** Two ways:

- **Option A (recommended):** Deploy the React build to chaioz.com.au — the `public/.well-known/*` files are served automatically.
- **Option B:** Add a reverse proxy rule on your domain so `chaioz.com.au/.well-known/*` → `https://late-night-chai-1.preview.emergentagent.com/api/well-known/*`. Cloudflare Workers / Nginx / AWS CloudFront can all do this in 2 lines of config.

### c) Verify

```bash
# These two URLs MUST return JSON when curl'd from the public internet:
curl https://chaioz.com.au/.well-known/apple-app-site-association
curl https://chaioz.com.au/.well-known/assetlinks.json
```

If they return JSON, restart your phone (yes, really — iOS aggressively caches AASA on the device) and tap any `https://chaioz.com.au/order/ABC123` link in Messages or Mail. It should open the Chaioz app.

## Step 5 — Invite beta testers

### TestFlight (iOS) — supports up to 10,000 testers
1. App Store Connect → TestFlight → **External Testing** → **+ Add a New Group** → "Chaioz Beta"
2. Add testers by email (max 10,000). They'll get an automated invite from Apple.
3. **First external build needs Apple's beta review** (~24h, lighter than App Store review). Subsequent updates skip review.

### Play Store internal track (Android) — up to 100 testers, instant
1. Play Console → **Testing → Internal testing** → **Create new release** (already done by `eas submit`)
2. **Testers** tab → add Google account emails / a Google Group
3. Send them the **opt-in URL** Play Console gives you — they install Chaioz from the Play Store (just shows "Internal" badge).

## Daily workflow (after step 1–4 are done)

```bash
# 1. Bump version in app.json (e.g. 0.1.0 → 0.1.1) — only needed if shipping a new build
# 2. Build + submit
cd /app/mobile
eas build --profile preview --platform all
eas submit --platform all --latest

# Testers get the new build automatically (TestFlight pushes; Play Store needs them to update from the Store)
```

OTA updates (no rebuild) for JS-only changes:

```bash
yarn global add eas-cli  # if not already
eas update --branch preview --message "Fixed checkout copy"
# Active testers get the update on next app open — no review, no rebuild.
```

## When you're ready to ship to the public App Store / Play Store

### App Store (iOS)
1. Bump version in `app.json` to `1.0.0`
2. `eas build --profile production --platform ios`
3. `eas submit --profile production --platform ios --latest`
4. App Store Connect → fill in screenshots + description + privacy nutrition labels → **Submit for Review**.
5. Review: usually 24–48 hours these days.

### Play Store (Android)
1. Bump version + `versionCode` in `app.json`
2. `eas build --profile production --platform android`
3. `eas submit --profile production --platform android`
4. Play Console → **Production** track → **Release management** → promote the build → fill in store listing → **Send for review**.
5. Review: usually 24–72 hours.

## Troubleshooting

| Symptom | Fix |
|---|---|
| Universal links open Safari instead of the app | (a) iOS device has cached old AASA — restart phone or wait 24h. (b) Verify `chaioz.com.au/.well-known/apple-app-site-association` returns JSON with **no `.json` extension and `Content-Type: application/json`** (not text/plain). (c) Make sure `ios.associatedDomains` in `app.json` lists `applinks:chaioz.com.au` exactly. |
| Push notifications don't arrive on TestFlight | (a) Open the app once and grant notification permission. (b) Check `/api/devices/register` was called — view backend logs `tail -f /var/log/supervisor/backend.err.log`. (c) Try `curl -X POST https://exp.host/--/api/v2/push/send -H "Content-Type: application/json" -d '[{"to":"ExponentPushToken[xxx]","title":"Test","body":"From curl"}]'` to isolate the backend vs the device. |
| `eas build` fails with "no provisioning profile" | Run `eas credentials --platform ios` and let EAS auto-generate a fresh one. |
| TestFlight says "Build not available" | Apple takes 5–15 min to process. Refresh in 10 min. |
